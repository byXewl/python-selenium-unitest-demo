PS:代码存在一些BUG，可能会产生一些scoped_dir开头的目录文件，自行搜索删除。
```
scoped_dir 文件夹通常是由使用 Selenium 测试框架和 ChromeDriver 时创建的，它们包含了浏览器的用户配置和其他临时数据 
。这些文件夹在测试执行结束后，理论上应该被自动删除。然而，在某些情况下，如果测试没有正常结束或者 driver.quit() 没有被正确调用，这些文件夹可能会残留在系统中 
。

在删除这些文件夹时需要谨慎，因为如果文件夹正在被使用，或者有相关的进程仍然在运行，直接删除可能会失败，甚至可能需要管理员权限 
。如果确定这些文件夹不再需要，可以安全地删除它们，以释放磁盘空间。但建议在删除之前，确保相关的 Selenium 测试脚本已经结束运行，并且所有的浏览器实例都已经被正确关闭 
。

如果你正在使用 Selenium 并遇到了大量 scoped_dir 文件夹堆积的问题，可以通过修改 Selenium 脚本，确保在测试结束后调用 driver.quit()，或者使用 driver.close() 来关闭浏览器窗口，这样可以有助于这些临时文件夹被自动清理 
```
